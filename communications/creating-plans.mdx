---
title: Creating communication plans
description: Set up structured communication strategies
---

Communication plans provide a framework for all stakeholder communications related to a change initiative.

## Creating a plan

1. Navigate to **Communications** in the sidebar
2. Click **New Plan**
3. Fill in plan details:

| Field | Description |
|-------|-------------|
| **Name** | Descriptive title (e.g., "Q1 ERP Launch Communications") |
| **Description** | Purpose and scope of the plan |
| **Project** | Associated change initiative |
| **Start date** | When communications begin |
| **End date** | When the campaign concludes |
| **Priority** | High, Medium, or Low |

4. Click **Create**

## Plan structure

Consider organizing your plan around change phases:

### Pre-change (Awareness)

- Announce the change
- Explain the reasons
- Share timeline and expectations

### During change (Enablement)

- Training notifications
- Progress updates
- Support resources

### Post-change (Sustainment)

- Success stories
- Ongoing support reminders
- Feedback collection

## Linking to projects

Each plan should link to a project:

- Activities can target project stakeholders
- Plan timeline aligns with project milestones
- Reports show communication coverage

## Plan settings

### Notifications

Configure notifications for:

- Upcoming activities
- Missed deadlines
- Completion milestones

### Permissions

Control who can:

- View the plan
- Edit activities
- Mark activities complete

## Templates

Create template plans for recurring communication patterns:

1. Build a plan with standard activities
2. Save as template
3. Duplicate for new projects
4. Adjust dates and details

<Tip>
  Start with more communication touchpoints than you think you need. It's easier to reduce frequency than to recover from under-communication.
</Tip>
