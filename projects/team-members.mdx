Project team members are users in your organization who can view and work on a specific project. Manage team access to control who sees project data and can make changes.

## Team member roles

Team members have one of these roles:

| Role | Permissions |
|------|-------------|
| **Owner** | Full access, can delete project and manage all settings |
| **Admin** | Edit all project data, manage team members |
| **Member** | View and edit project content, cannot manage settings |
| **Viewer** | Read-only access to project data |

## Adding team members

1. Open the project
2. Go to **Settings** → **Team**
3. Click **Add Member**
4. Search for the user by name or email
5. Select their role
6. Click **Add**

<Note>
  Users must be members of your organization before they can be added to a project. Add them in **Settings** → **Team Management** first.
</Note>

## Removing team members

1. Go to **Settings** → **Team**
2. Find the member to remove
3. Click the **Remove** button
4. Confirm the removal

Removed members lose access immediately but any work they created remains.

## Changing roles

1. Go to **Settings** → **Team**
2. Find the member
3. Click their current role
4. Select the new role
5. Changes apply immediately

## Ownership transfer

Transfer project ownership to another admin:

1. Go to **Settings** → **Team**
2. Find the new owner
3. Click **Make Owner**
4. Confirm the transfer

You must be the current owner to transfer ownership.

## Default access

Organization admins automatically have access to all projects. This cannot be changed.

Organization members need to be explicitly added to projects to gain access.
