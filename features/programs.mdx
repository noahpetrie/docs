Programs allow you to organize multiple related change projects under a single umbrella. Use programs for large-scale transformations that span multiple initiatives.

## What is a program?

A program is a collection of related projects that share:

- Common business objectives
- Shared stakeholders or resources
- Interdependencies between changes
- Unified governance and reporting

**Example**: A "Digital Transformation" program might include:
- CRM implementation project
- Cloud migration project  
- Process automation project

## Creating a program

<Note>
  Program creation requires admin permissions.
</Note>

1. Navigate to **Programs** from the sidebar
2. Click **New Program**
3. Fill in program details:
   - **Name** - Descriptive program title
   - **Description** - Purpose and scope
   - **Start date** - When the program begins
   - **Target date** - Expected completion
4. Click **Create**

## Program dashboard

The program view shows:

- **Project count** - Number of linked projects
- **Overall status** - Aggregated from project statuses
- **Timeline** - Program duration with project milestones
- **Stakeholder overlap** - People affected by multiple projects

## Linking projects to programs

### From the program

1. Open the program
2. Click **Add Projects**
3. Select existing projects to include
4. Save changes

### From a project

1. Open project settings
2. Select **Program** dropdown
3. Choose the parent program
4. Save

## Cross-project insights

Programs provide portfolio-level visibility:

### Shared stakeholders

Identify stakeholders affected by multiple projects:

- Avoid change fatigue by coordinating communications
- Recognize cumulative impact burden
- Plan integrated engagement strategies

### Impact overlap

See where changes intersect:

- Technology changes affecting the same systems
- Process changes in the same departments
- Training needs that can be combined

### Timeline coordination

Visualize all project timelines together:

- Avoid scheduling conflicts
- Stagger go-lives to reduce risk
- Identify resource bottlenecks

## Managing programs

### Program status

Program status is calculated from project statuses:

| Condition | Program status |
|-----------|----------------|
| All projects completed | Completed |
| Any project on hold | At Risk |
| All projects active | In Progress |
| No projects started | Planning |

### Removing projects

To remove a project from a program:

1. Open the program
2. Find the project in the list
3. Click **Remove from Program**

The project continues to exist independently.

## Best practices

<AccordionGroup>
  <Accordion title="Define clear program scope">
    Only group projects that genuinely share objectives or dependencies. Avoid creating programs just for organizational convenience.
  </Accordion>
  <Accordion title="Assign a program manager">
    Designate someone responsible for cross-project coordination and reporting.
  </Accordion>
  <Accordion title="Regular program reviews">
    Hold periodic reviews to assess overall program health and address cross-cutting issues.
  </Accordion>
