Projects are the core organizational unit in Leap Change. Each project represents a distinct change initiative and contains all related stakeholders, change impacts, surveys, and communication plans.

## Project overview

Projects help you track and manage individual change efforts within your organization. They provide a central location for:

- Stakeholder management and engagement tracking
- Change impact identification and mitigation
- Survey distribution and readiness assessment
- Communication planning and execution

## Project statuses

Projects can be in one of three statuses:

| Status | Description |
|--------|-------------|
| **Active** | Ongoing change initiatives currently being managed |
| **On Hold** | Temporarily paused projects |
| **Archived** | Completed or cancelled projects (preserved for reference) |

## Creating a project

1. Navigate to **Projects** in the sidebar
2. Click **New Project**
3. Complete the guided wizard:
   - **Describe the change** - Name, description, owner, sponsor
   - **Classify** - Change type, priority, departments affected
   - **Set timeline** - Start date, target go-live, milestones
   - **Transition planning** - Communication, training, support approach
4. Click **Create** to launch your project

## Project views

### Grid view

Display projects as cards showing:

- Project name and description
- Status indicator
- Stakeholder count
- Change impact count

### List view

Table format with sortable columns:

- Name
- Status
- Start date
- Target date
- Stakeholders
- Impacts

## Managing projects

### Filtering

Filter the project list by:

- **Status** - Show active, on hold, or archived projects
- **Search** - Find projects by name

### Project details

Click on any project to access:

- Project overview and settings
- Associated stakeholders
- Linked change impacts
- Communication plans
- Survey results

### Archiving projects

Archive completed or cancelled projects:

1. Open the project
2. Click the archive icon
3. Confirm the action

<Note>
  Archived projects are preserved and can be restored. Use the status filter to view archived projects.
</Note>

### Deleting projects

<Warning>
  Deleting a project permanently removes all associated data including stakeholder links and change impacts.
</Warning>

1. Open the project
2. Click delete
3. Review the impact count warning
4. Confirm deletion

## Project metrics

Each project displays key metrics:

- **Stakeholder count** - Number of people linked to the project
- **Impact count** - Number of change impacts identified
- **Completion status** - Progress through project lifecycle

## Best practices

<AccordionGroup>
  <Accordion title="Use clear naming conventions">
    Include the year and change type in project names (e.g., "2026 ERP Migration - Finance Module") for easy identification.
  </Accordion>
  <Accordion title="Assign ownership early">
    Define the change owner and executive sponsor before starting stakeholder engagement.
  </Accordion>
  <Accordion title="Link to programs">
    Group related projects under programs to identify overlapping impacts and coordinate communications.
  </Accordion>
  <Accordion title="Regular status updates">
    Keep project status current to ensure accurate reporting and visibility.
  </Accordion>
</AccordionGroup>
