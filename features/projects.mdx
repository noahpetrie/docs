Projects are the foundation of change management in Leap Change. Each project represents a distinct change initiative and contains all related stakeholders, impacts, surveys, and communications.

## Creating a project

1. Navigate to **Projects** from the sidebar
2. Click **New Project**
3. Follow the guided wizard:
   - **Describe the change** - What is changing and why
   - **Classify** - Change type and priority
   - **Set timeline** - Start and target dates
   - **Plan transition** - Communication and training approach
4. Click **Create** to launch your project

## Project properties

| Property | Description |
|----------|-------------|
| **Name** | Clear, descriptive project title |
| **Description** | Detailed explanation of the change |
| **Status** | Active, On Hold, Completed, or Archived |
| **Priority** | Critical, High, Medium, or Low |
| **Change type** | Technology, Process, Organizational, Policy |
| **Start date** | When change activities begin |
| **Target date** | Expected implementation date |

## Project dashboard

Each project has a dashboard showing:

- **Stakeholder count** - People affected by this project
- **Impact count** - Change impacts being tracked
- **Survey responses** - Readiness assessment results
- **Communication activities** - Planned and completed touchpoints

## Managing projects

### Project members

Assign team members to projects:

1. Open the project
2. Go to **Members** tab
3. Click **Add Member**
4. Select team members and their project roles

### Archiving projects

When a project is complete:

1. Open project settings
2. Click **Archive Project**
3. Archived projects are preserved but hidden from active views

Archived projects can be restored at any time.

## Best practices

<AccordionGroup>
  <Accordion title="Use clear naming conventions">
    Include the year and change type in project names (e.g., "2026 ERP Migration - Finance Module") for easy identification.
  </Accordion>
  <Accordion title="Set realistic timelines">
    Factor in assessment, planning, and sustainment phases—not just implementation.
  </Accordion>
  <Accordion title="Assign ownership early">
    Define the change owner and sponsor before starting stakeholder engagement.
  </Accordion>
