Leap Change supports keyboard shortcuts for faster navigation and common actions.

## Viewing shortcuts

Press **?** anywhere in the app to open the keyboard shortcuts panel.

## Navigation shortcuts

Use these shortcuts to quickly navigate between sections:

| Shortcut | Action |
|----------|--------|
| `g` then `h` | Go to Home |
| `g` then `s` | Go to Stakeholders |
| `g` then `p` | Go to Projects |
| `g` then `c` | Go to Change Impacts |
| `g` then `v` | Go to Surveys |
| `g` then `k` | Go to Kanban board |
| `g` then `r` | Go to Reports |
| `g` then `t` | Go to Settings |

<Tip>
  Navigation shortcuts use a two-key sequence. Press `g` first, release, then press the second key.
</Tip>

## Global shortcuts

These shortcuts work from anywhere in the app:

| Shortcut | Action |
|----------|--------|
| `⌘` + `K` | Open command palette |
| `⌘` + `.` | Toggle sidebar |
| `?` | Show keyboard shortcuts |

On Windows, use `Ctrl` instead of `⌘`.

## Command palette

The command palette (`⌘ K`) provides quick access to:

- **Navigation** - Jump to any page
- **Actions** - Create stakeholders, impacts, tasks
- **Search** - Find content across your workspace
- **Settings** - Quick access to preferences

### Using the command palette

1. Press `⌘ K` to open
2. Start typing to search
3. Use arrow keys to navigate results
4. Press `Enter` to select
5. Press `Escape` to close

## Platform differences

| Mac | Windows/Linux |
|-----|---------------|
| `⌘` (Command) | `Ctrl` |
| `⌥` (Option) | `Alt` |

## Tips for efficiency

<AccordionGroup>
  <Accordion title="Learn the navigation shortcuts first">
    The `g` + key pattern is consistent and easy to remember. Start with the pages you use most often.
  </Accordion>
  <Accordion title="Use the command palette for everything else">
    If you forget a shortcut, the command palette can do almost anything. Press `⌘ K` and type what you want to do.
  </Accordion>
  <Accordion title="Press ? when stuck">
    The shortcuts panel shows all available shortcuts for your current view.
  </Accordion>
</AccordionGroup>
