title: Impact report
description: Analyze change impact severity and completion status
---

The Impact Report provides visibility into your change impacts—their severity distribution, completion status, and trends over time.

## Report sections

### Severity distribution

Pie chart showing impacts by severity:

- **High** - Major impacts requiring significant attention
- **Medium** - Moderate impacts with manageable effort
- **Low** - Minor impacts with minimal disruption

Use this to:
- Assess overall change burden
- Identify high-risk periods
- Allocate resources appropriately

### Status breakdown

View impacts by lifecycle status:

- Backlog
- Todo
- In Progress
- In Review
- Done
- Canceled
- Duplicate

Track completion trends and identify bottlenecks.

### Timeline view

See impacts over time:

- When impacts were identified
- Mitigation progress
- Expected completion dates
- Actual completion dates

### Category analysis

Break down impacts by category:

- Process changes
- Technology changes
- Skills/knowledge requirements
- Organizational structure
- Policy/compliance
- Culture changes

## Using the report

### Filtering

Narrow the report scope:

- **Project** - One or multiple projects
- **Date range** - Created or due within dates
- **Severity** - Focus on specific severity levels
- **Status** - Exclude completed or show only active

### Drill down

Click any chart segment to:

- See the underlying impacts
- Navigate to individual impact details
- Apply as a filter to other sections

### Export

Export for presentations:

- **PDF summary** - Executive overview
- **Detailed CSV** - All impact data
- **Chart images** - Individual visualizations

## Insights

The report helps answer:

- How many high-severity impacts do we have?
- Are we completing impacts on schedule?
- Which categories have the most impacts?
- What's our impact backlog size?
- Are we identifying impacts early enough?

<Tip>
  Review the Impact Report weekly during active change projects to catch emerging patterns and address bottlenecks early.
</Tip>
