title: Tasks and Kanban board
description: Manage change management work items

The Tasks feature provides a Kanban board for managing change management activities. Track work items through their lifecycle from planning to completion.

## Kanban board

Access the Kanban board from **Tasks** in the sidebar.

### Columns

Tasks move through these stages:

| Column | Description |
|--------|-------------|
| **Planned** | Identified but not yet ready to start |
| **Up Next** | Ready to begin work |
| **In Progress** | Currently being worked on |
| **In Review** | Completed, awaiting validation |
| **Done** | Successfully completed |
| **Canceled** | No longer needed |

### Moving tasks

Drag and drop tasks between columns:

1. Click and hold a task card
2. Drag to the target column
3. Drop to update status

Tasks can also be moved via the task detail view.

### Reordering tasks

Drag tasks within a column to change their priority order. Tasks higher in the column indicate higher priority.

## Creating tasks

### Quick add

1. Click **+ Add Task** at the top of any column
2. Enter the task title
3. Press Enter to create

The task is created in that column with default settings.

### Detailed creation

1. Click **New Task** button
2. Fill in task details:
   - **Title** - What needs to be done
   - **Description** - Additional context and details
   - **Project** - Associated change initiative
   - **Assignee** - Who is responsible
   - **Due date** - When it should be completed
   - **Priority** - High, Medium, Low
   - **Labels** - Tags for categorization

## Task properties

### Assignee

Assign tasks to team members:

- Each task has one assignee
- Assignee receives notifications for updates
- Filter board by assignee to see your work
- Assignee avatar appears on task card

### Priority

Set task urgency:

| Priority | Use case |
|----------|----------|
| **High** | Needs immediate attention, blocking other work |
| **Medium** | Standard priority, complete within normal timeline |
| **Low** | Can wait if needed, nice-to-have |

### Labels

Add labels for categorization:

- Create custom labels with colors
- Apply multiple labels per task
- Filter board by label
- Examples: "Training", "Communication", "Technical", "Urgent"

### Due dates

Set deadlines:

- Overdue tasks highlighted in red
- Due soon (within 3 days) highlighted in yellow
- Filter by due date range
- Sort tasks by due date

## Filtering and views

### Filter options

Filter the board by:

- **Assignee** - Show tasks for specific team members
- **Project** - Show tasks for a specific initiative
- **Priority** - Filter by urgency
- **Labels** - Filter by categorization
- **Due date range** - Show tasks due within a period

### Search

Search tasks by title or description to quickly find specific work items.

### Saved filters

Save commonly-used filter combinations for quick access.

## Task details

Click any task card to open the full detail view:

- **Activity log** - History of changes
- **Comments** - Discussion thread
- **Attachments** - Related files
- **Subtasks** - Break down into smaller items
- **Related impacts** - Linked change impacts

## Best practices

<AccordionGroup>
  <Accordion title="Keep task titles actionable">
    Start with a verb: "Create training materials" rather than "Training materials."
  </Accordion>
  <Accordion title="Limit work in progress">
    Don't have too many tasks "In Progress" at once. Complete current work before starting new items.
  </Accordion>
  <Accordion title="Use labels consistently">
    Define a standard set of labels for your team and use them consistently across all tasks.
  </Accordion>
  <Accordion title="Review regularly">
    Hold regular reviews to move completed tasks, identify blockers, and prioritize the backlog.
  </Accordion>
  <Accordion title="Link to change impacts">
    Connect tasks to the change impacts they address for better traceability.
  </Accordion>
</AccordionGroup>
