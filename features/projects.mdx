Projects are the core organizational unit in Leap Change. Each project represents a distinct change initiative and contains all related stakeholders, impacts, surveys, communications, and tasks.

## Understanding projects

A project in Leap Change represents a specific change effort:

- A technology implementation
- A process improvement
- An organizational restructure
- A policy or compliance change
- A cultural transformation

## Project properties

Each project includes:

| Property | Description |
|----------|-------------|
| **Name** | Clear identifier for the initiative |
| **Description** | Detailed explanation of what's changing and why |
| **Status** | Active, On Hold, Completed, or Archived |
| **Priority** | Critical, High, Medium, or Low |
| **Change type** | Technology, Process, Organizational, Policy, etc. |
| **Timeline** | Start date and target go-live date |
| **Owner** | Person responsible for the change |
| **Sponsor** | Executive sponsor supporting the initiative |

## Project lifecycle

Projects move through these stages:

1. **Active** - Currently being worked on
2. **On Hold** - Temporarily paused
3. **Completed** - Successfully implemented
4. **Archived** - Closed and removed from active views

## Managing projects

### Viewing projects

1. Navigate to **Projects** in the sidebar
2. See all projects in a list view
3. Filter by status, priority, or type
4. Click any project to open its details

### Editing projects

1. Open the project
2. Click **Settings** or the edit icon
3. Update any project properties
4. Save changes

### Project members

Add team members to work on the project:

1. Open project settings
2. Go to **Members**
3. Click **Add Member**
4. Select team members from your organization
5. Define their role on the project

### Archiving projects

When a project is complete:

1. Open project settings
2. Click **Archive Project**
3. Confirm the action

Archived projects:
- Are removed from active views
- Retain all historical data
- Can be restored if needed

## Programs

Group related projects into programs for coordinated management:

- Shared strategic objectives
- Overlapping stakeholders
- Interdependent timelines
- Consolidated reporting
