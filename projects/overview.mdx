Projects are the central organizational unit in Leap Change. Each project represents a distinct change initiative with its own stakeholders, impacts, surveys, and communication plans.

## What is a project?

A project contains everything needed to manage a change initiative:

- **Stakeholders** - People affected by the change
- **Change impacts** - Specific effects of the change
- **Surveys** - Feedback and readiness assessments
- **Communications** - Planned stakeholder communications
- **Tasks** - Work items to complete

## Project properties

### Core details

| Field | Description |
|-------|-------------|
| **Name** | Descriptive project title |
| **Description** | Detailed explanation of the change |
| **Change owner** | Person responsible for the initiative |
| **Sponsor** | Executive supporting the change |

### Classification

| Field | Description |
|-------|-------------|
| **Change type** | Technology, Process, Organizational, Policy |
| **Priority** | Critical, High, Medium, Low |
| **Departments** | Affected business areas |
| **Geographic scope** | Local, Regional, or Global |

### Timeline

| Field | Description |
|-------|-------------|
| **Start date** | When change activities begin |
| **Target go-live** | Expected implementation date |
| **Status** | Active, On Hold, Completed, Archived |

## Project statuses

Projects can have the following statuses:

- **Active** - Currently in progress
- **On Hold** - Temporarily paused
- **Completed** - Successfully finished
- **Archived** - No longer active, preserved for reference

## Accessing projects

1. Navigate to **Projects** in the sidebar
2. View all projects in a list or card view
3. Click a project to open its detail page

### Project dashboard

Each project has a dashboard showing:

- Key metrics (stakeholder count, impact count, survey responses)
- Recent activity
- Upcoming tasks and milestones
- Team members
