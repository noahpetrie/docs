Projects (also called initiatives) are the core organizational unit in Leap Change. Each project represents a specific change effort and contains all related stakeholders, impacts, surveys, and communication plans.

## Creating a project

1. Navigate to **Projects** in the sidebar
2. Click **New Project**
3. Follow the guided wizard:

### Step 1: Describe the change

- **Project name**: Clear, descriptive title
- **Description**: What is changing and why
- **Change owner**: Person responsible for the change
- **Sponsor**: Executive sponsor

### Step 2: Classify

| Field | Description |
|-------|-------------|
| **Change type** | Technology, Process, Organizational, Policy |
| **Priority** | Critical, High, Medium, Low |
| **Departments** | Which areas are affected |
| **Geographic scope** | Local, Regional, Global |

### Step 3: Set timeline

- **Start date**: When change activities begin
- **Target go-live**: Expected implementation date
- **Key milestones**: Important checkpoints

### Step 4: Plan transition

- **Communication strategy**: How you'll inform stakeholders
- **Training approach**: How you'll prepare people
- **Support plan**: How you'll help during transition

### Step 5: Review and launch

Review all details and click **Create Project** to launch.

## Project views

### Grid view

Display projects as cards showing:

- Project name and status
- Stakeholder count
- Impact count
- Timeline progress

### List view

Table format with sortable columns for detailed analysis.

## Project properties

### Status

Track project lifecycle:

| Status | Description |
|--------|-------------|
| **Active** | Currently in progress |
| **On hold** | Temporarily paused |
| **Completed** | Successfully finished |
| **Archived** | Closed for reference |

### Priority

Set urgency level:

- **Critical**: Immediate executive attention required
- **High**: Significant organizational importance
- **Medium**: Standard project priority
- **Low**: Can proceed at slower pace

### Timeline

- **Start date**: When planning or execution begins
- **Target end date**: Expected completion
- **Actual end date**: When project completed (set automatically)

## Project dashboard

Each project has a dedicated dashboard showing:

- **Overview metrics**: Stakeholder count, impact summary
- **Recent activity**: Latest changes and updates
- **Quick actions**: Common tasks
- **Progress indicators**: Timeline and completion status

## Linking to programs

Associate projects with programs for portfolio management:

1. Open project settings
2. Select **Program** field
3. Choose the parent program
4. Save changes

## Project team

Manage who has access:

- **Owner**: Full control, can delete project
- **Team members**: Organization members with access
- **Viewers**: Read-only access (coming soon)

## Archiving projects

When a project is complete:

1. Open project settings
2. Click **Archive Project**
3. Confirm the action

Archived projects:

- Remain accessible for reference
- Don't appear in active project lists
- Can be restored if needed

## Deleting projects

<Warning>
  Deleting a project permanently removes all associated data including stakeholder links, impacts, surveys, and communications. This cannot be undone.
</Warning>

Only project owners and organization admins can delete projects.

## Filtering projects

Use filters to find specific projects:

- **Status**: Active, on hold, completed, archived
- **Program**: Group by parent program
- **Priority**: Filter by urgency
- **Search**: Find by name or description
