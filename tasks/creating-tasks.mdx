---
title: Creating tasks
description: Create and manage tasks for your change initiatives
---

Tasks help you track action items, assignments, and deliverables for your change initiatives.

## Creating a new task

1. Navigate to **Tasks** in the sidebar
2. Click **Add task** or the **+** button in any column
3. Fill in task details
4. Click **Create**

## Task fields

### Required fields

| Field | Description |
|-------|-------------|
| **Title** | Brief task description |
| **Status** | Current workflow state |

### Optional fields

| Field | Description |
|-------|-------------|
| **Description** | Detailed task information |
| **Assignee** | Team member responsible |
| **Project** | Associated change initiative |
| **Priority** | Task urgency level |
| **Due date** | Target completion date |
| **Labels** | Tags for categorization |

## Task status

Tasks progress through these stages:

| Status | Description |
|--------|-------------|
| **Planned** | Identified but not started |
| **Up next** | Prioritized for upcoming work |
| **In progress** | Currently being worked on |
| **In review** | Completed, awaiting review |
| **Done** | Successfully completed |
| **Canceled** | No longer needed |

## Priority levels

Set task urgency:

| Priority | Use for |
|----------|----------|
| **High** | Critical path items, blockers |
| **Medium** | Important but not urgent |
| **Low** | Nice to have, backlog items |

## Assigning tasks

Assign tasks to team members:

1. Open the task
2. Click the **Assignee** field
3. Search for a team member
4. Select their name
5. Save the task

Assignees receive notifications when:
- They are assigned a task
- Task due date approaches
- Task status changes

## Linking to projects

Associate tasks with change initiatives:

1. Open the task
2. Click the **Project** field
3. Select the relevant project
4. Save changes

Linked tasks appear in:
- Project task lists
- Project reports
- Kanban board filters

## Using labels

Labels help categorize and filter tasks:

1. Open the task
2. Click **Add label**
3. Select existing labels or create new ones
4. Save the task

Common label strategies:
- Category (Communication, Training, Technical)
- Team (HR, IT, Operations)
- Sprint or phase
- Blockers and dependencies

## Best practices

<Tip>
  Keep task titles concise but descriptive. Use the description field for details.
</Tip>

### Clear ownership
- Assign one owner per task
- Ensure assignees have capacity
- Reassign promptly when needed

### Realistic dates
- Set achievable due dates
- Account for dependencies
- Update dates when plans change

### Regular updates
- Move tasks through stages promptly
- Add comments for context
- Close completed tasks
