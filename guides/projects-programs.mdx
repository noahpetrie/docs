Leap Change uses a two-level hierarchy to organize change work: programs contain projects, and projects contain the day-to-day change management activities.

## Understanding the hierarchy

```
Program (Digital Transformation)
├── Project (CRM Implementation)
│   ├── Stakeholders
│   ├── Change Impacts
│   ├── Surveys
│   └── Communications
├── Project (Process Automation)
│   └── ...
└── Project (Cloud Migration)
    └── ...
```

## When to use programs

Create a program when you have:

- Multiple related change initiatives
- Shared stakeholders across projects
- Interdependencies between projects
- Need for portfolio-level reporting

## When to use standalone projects

Use standalone projects (without a program) for:

- Single, isolated change initiatives
- Pilot projects
- Small-scale changes
- Projects that don't relate to other work

## Creating a program

1. Navigate to **Programs** in the sidebar
2. Click **New Program**
3. Enter:
   - **Name**: Clear identifier (e.g., "Digital Transformation 2026")
   - **Description**: Program objectives and scope
4. Click **Create**

## Creating a project

1. Navigate to **Projects** in the sidebar
2. Click **New Project**
3. Enter the project name
4. Optionally select a parent program
5. Click **Create**

## Linking projects to programs

To add an existing project to a program:

1. Open the project
2. Click **Settings** or the edit icon
3. In the **Program** field, select the program
4. Save changes

To remove from a program:

1. Open the project
2. Clear the **Program** field
3. Save changes

## Program dashboard

The program detail page shows:

| Metric | Description |
|--------|-------------|
| Projects | Total initiatives in the program |
| Stakeholders | Unique stakeholders across all projects |
| Change impacts | Combined impacts across projects |
| Overall status | Aggregate health indicator |

## Stakeholder overlap analysis

Programs help identify stakeholders affected by multiple projects:

1. Open a program
2. View the **Stakeholder Overlap** section
3. See which stakeholders appear in multiple projects
4. Use this for coordinated communication planning

## Project lifecycle

| Status | When to use |
|--------|-------------|
| **Active** | Work is ongoing |
| **On Hold** | Temporarily paused |
| **Completed** | All work finished successfully |
| **Archived** | Historical record, no longer active |

## Deleting projects and programs

<Warning>
  Deleting a project removes all associated stakeholders, impacts, surveys, and communications. This cannot be undone.
</Warning>

### Deleting a project

1. Open the project
2. Go to **Settings** → **Danger Zone**
3. Click **Delete Project**
4. Type the project name to confirm
5. Click **Delete**

### Deleting a program

1. Open the program
2. Click **Delete**
3. Confirm deletion

<Note>
  Deleting a program does not delete its projects. Projects become standalone.
</Note>
