Programs help you organize and manage multiple related change initiatives under a single portfolio. Use programs when running large-scale transformations that span multiple projects.

## What is a program?

A **program** is a collection of related change initiatives that share common goals, timelines, or organizational impact. Programs provide:

- **Portfolio visibility** - See all related projects in one place
- **Consolidated reporting** - Aggregate metrics across initiatives
- **Coordinated planning** - Identify dependencies and overlapping impacts
- **Stakeholder alignment** - Track people affected by multiple changes

## When to use programs

Create a program when you have:

- Multiple related change initiatives
- Shared stakeholders across projects
- Common timelines or dependencies
- Need for executive-level reporting

### Example: Digital transformation program

A digital transformation program might contain:

- CRM implementation project
- ERP migration project
- Cloud infrastructure modernization
- Digital skills training initiative

## Program properties

| Field | Description |
|-------|-------------|
| **Name** | Program identifier |
| **Description** | Purpose and scope |
| **Status** | Active, On-hold, or Completed |
| **Start date** | When the program begins |
| **Target end date** | Expected completion |
| **Projects** | Linked change initiatives |

## Creating a program

1. Navigate to **Programs** in the sidebar
2. Click **New Program**
3. Enter program details:
   - Name and description
   - Status
   - Date range
4. Click **Create**

## Program dashboard

The program detail page shows:

- **Project count** - Number of linked initiatives
- **Combined stakeholders** - Total people affected
- **Impact summary** - Aggregated change impacts
- **Timeline** - Program-level milestones
- **Status overview** - Project health indicators

## Best practices

<AccordionGroup>
  <Accordion title="Define clear boundaries">
    Programs should have a clear scope. Avoid creating programs that are too broad or contain unrelated initiatives.
  </Accordion>
  <Accordion title="Assign program leadership">
    Every program needs a program manager who oversees coordination across projects.
  </Accordion>
  <Accordion title="Regular reviews">
    Schedule periodic program reviews to assess progress and identify cross-project dependencies.
  </Accordion>
  <Accordion title="Communicate at the program level">
    Some communications should address the overall transformation, not just individual projects.
  </Accordion>
</AccordionGroup>
