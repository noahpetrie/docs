Projects are the foundation of change management in Leap Change. Each project represents a distinct change initiative and contains all related stakeholders, impacts, surveys, tasks, and communications.

## Understanding projects

A project in Leap Change represents a single change initiative. Projects help you:

- **Organize change activities** - Keep related work together
- **Track stakeholders** - Know who is affected
- **Document impacts** - Understand what's changing
- **Measure readiness** - Assess preparedness
- **Plan communications** - Coordinate messaging

## Project properties

### Core fields

| Field | Description |
|-------|-------------|
| **Name** | Clear, descriptive project title |
| **Description** | Detailed explanation of the change |
| **Status** | Active, On Hold, Completed, or Archived |
| **Priority** | Critical, High, Medium, or Low |
| **Change type** | Technology, Process, Organizational, Policy |

### Timeline

- **Start date** - When change activities begin
- **Target end date** - Expected completion date

### Relationships

- **Program** - Optional parent program grouping related projects
- **Stakeholder count** - Number of linked stakeholders
- **Impact count** - Number of documented change impacts

## Accessing projects

Projects are accessed through the **Dashboard** section in the sidebar. The dashboard shows:

- All active projects in card view
- Archived projects in a separate section
- Quick metrics for each project (stakeholders, impacts)
- Project status badges

## Creating a project

1. Navigate to the Dashboard
2. Click **New Project**
3. Complete the guided wizard:
   - Describe the change
   - Classify the project type and priority
   - Set timeline and milestones
   - Plan transition approach
4. Click **Create** to launch

<Info>
  The project wizard walks you through all required information step by step. See [Create your first project](/getting-started/first-project) for a detailed walkthrough.
</Info>

## Managing projects

### Editing a project

1. Open the project from the dashboard
2. Click the project name or settings icon
3. Update fields as needed
4. Changes save automatically

### Project statuses

| Status | Description |
|--------|-------------|
| **Active** | Currently in progress |
| **On Hold** | Temporarily paused |
| **Completed** | Successfully finished |
| **Archived** | No longer active, data preserved |

### Archiving projects

Archive completed or cancelled projects:

1. Open the project
2. Change status to **Archived**
3. Project moves to the archived section

<Note>
  Archiving preserves all project data. You can restore archived projects at any time.
</Note>

## Project team

### Member roles

Project members can have different roles:

- **Owner** - Full control over project settings
- **Admin** - Can edit all project data
- **Member** - Can view and contribute

### Adding team members

1. Open the project settings
2. Go to the Team section
3. Click **Add Member**
4. Select the user and assign a role

## Working within a project

Once inside a project, you have access to:

- **Stakeholders** - People affected by this change
- **Change Impacts** - Documented effects of the change
- **Surveys** - Readiness assessments for this project
- **Communications** - Plans linked to this project
- **Tasks** - Work items associated with this project

## Project insights

Each project dashboard shows:

- Stakeholder count and distribution
- Impact severity breakdown
- Survey response rates
- Communication completion status
- Task progress

## Best practices

<AccordionGroup>
  <Accordion title="Use clear naming conventions">
    Include the year and change type (e.g., "2026 ERP Migration - Finance Module") for easy identification.
  </Accordion>
  <Accordion title="Set realistic timelines">
    Factor in assessment, planning, implementation, and sustainment phases.
  </Accordion>
  <Accordion title="Assign ownership early">
    Define the change owner and sponsor before starting stakeholder engagement.
  </Accordion>
  <Accordion title="Keep projects focused">
    Create separate projects for distinct changes. Use programs to group related projects.
  </Accordion>
</AccordionGroup>
