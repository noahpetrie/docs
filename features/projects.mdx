Projects (also called initiatives) are the core organizational unit for change management in Leap Change. Each project represents a specific change effort and contains stakeholders, impacts, surveys, and communication plans.

## Creating a project

1. Navigate to **Projects** in the sidebar
2. Click **New Project**
3. Complete the guided wizard:

### Step 1: Describe the change

- **Project name** - Clear, descriptive title
- **Description** - Detailed explanation of what's changing and why
- **Change owner** - Person responsible for the change
- **Sponsor** - Executive sponsor supporting the initiative

### Step 2: Classify

| Field | Options |
|-------|---------|
| **Change type** | Technology, Process, Organizational, Policy, Culture |
| **Priority** | Critical, High, Medium, Low |
| **Departments** | Which business areas are affected |
| **Geographic scope** | Local, Regional, Global |

### Step 3: Timeline

- **Start date** - When change activities begin
- **Target go-live** - Expected implementation date
- **Milestones** - Key checkpoints (optional)

### Step 4: Transition planning

- **Communication strategy** - How you'll keep stakeholders informed
- **Training approach** - How you'll prepare people
- **Support plan** - How you'll help during transition

### Step 5: Launch

Review your settings and click **Create Project**.

## Project dashboard

Each project has a dashboard showing:

- **Stakeholder count** - Number of people involved
- **Impact summary** - High/medium/low impact breakdown
- **Survey status** - Active surveys and response rates
- **Communication progress** - Activities completed vs. planned
- **Task completion** - Work items status

## Managing projects

### Editing projects

1. Open the project
2. Click the settings icon or **Edit**
3. Update name, description, timeline, or other details
4. Save changes

### Project status

Projects can have these statuses:

| Status | Description |
|--------|-------------|
| **Active** | Currently in progress |
| **On-hold** | Temporarily paused |
| **Completed** | Successfully finished |

### Archiving projects

Archive completed projects to keep your workspace organized:

1. Open project settings
2. Click **Archive**
3. Confirm the action

Archived projects are preserved with all data but hidden from main views. You can restore them later if needed.

### Deleting projects

<Warning>
  Deleting a project permanently removes all associated data including stakeholder assignments, change impacts, surveys, and communications. This cannot be undone.
</Warning>

## Filtering and search

Find projects quickly:

- **Search** - Filter by project name
- **Status filter** - Show active, on-hold, or completed projects
- **Sort** - By name, date created, or last updated

## Project permissions

All organization members can:

- View projects
- Edit stakeholders and impacts
- Create surveys and communications

Organization admins can additionally:

- Archive or delete projects
- Manage project membership (if enabled)

## Best practices

<AccordionGroup>
  <Accordion title="Use clear naming conventions">
    Include year and change type in project names (e.g., "2026 ERP Migration - Finance Module").

    Factor in assessment, planning, implementation, and sustainment phases.
