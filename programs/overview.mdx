Programs help you group related change initiatives together, enabling coordinated planning and consolidated reporting across multiple projects.

## What is a program?

A program is a collection of related initiatives that share:

- Common strategic objectives
- Overlapping stakeholder groups
- Interdependencies or sequencing requirements
- Shared resources or timelines

**Example**: A "Digital transformation" program might include:
- New CRM implementation
- Legacy system retirement
- Process automation initiative
- Digital skills training

## Program stages

Programs progress through these stages:

| Stage | Description |
|-------|-------------|
| **Initiation** | Program defined and approved |
| **Planning** | Detailed planning in progress |
| **Execution** | Active implementation underway |
| **Adoption** | Changes being adopted by stakeholders |
| **Sustainment** | Changes embedded in operations |

## Program properties

Each program includes:

### Core information
- **Title** - Program name (up to 150 characters)
- **Description** - Brief overview of objectives and scope
- **Unique ID** - Auto-generated identifier

### Ownership
- **Change owner** - Person accountable for the program
- **Sponsor** - Executive sponsor providing oversight
- **Departments** - Business units involved

### Timeline
- **Planned start date** - When the program begins
- **Go-live date** - Target completion date

### Classification
- **Category** - Type of change (Technology, Process, Organizational, etc.)
- **Geographic scope** - Regions affected
- **Impact severity** - Overall impact level
- **Audience size** - Number of people affected

### Additional details
- **Dependencies** - External factors or prerequisites
- **Tags** - Labels for filtering and organization

## Program-level insights

Programs provide consolidated views across their initiatives:

### Stakeholder overlap

Identify stakeholders affected by multiple initiatives within the program. This helps:

- Prevent communication fatigue
- Coordinate messaging timing
- Understand cumulative change burden

### Impact aggregation

See the combined change impact across all program initiatives:

- Total impacts by severity
- Impacts by category
- Cross-initiative dependencies

### Timeline coordination

Visualize initiative timelines together to:

- Identify scheduling conflicts
- Optimize sequencing
- Balance resource demands
