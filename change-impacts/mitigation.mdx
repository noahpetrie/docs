---
title: Mitigation planning
description: Plan and track actions to address change impacts
---

Each change impact can have a mitigation plan with specific action items to help stakeholders prepare for and adopt the change.

## Creating a mitigation plan

1. Open a change impact
2. Scroll to the **Mitigation** section
3. Add plan details:
   - **Mitigation plan** - Overall approach description
   - **Start date** - When mitigation activities begin
   - **Expected completion** - Target date for completion

## Mitigation items

Break down your plan into specific action items:

### Adding items

1. In the Mitigation section, click **Add Item**
2. Enter the action description
3. Set status: **Todo**, **In Progress**, or **Done**
4. Assign an owner (optional)
5. Set a due date (optional)

### Managing items

- **Check off** completed items to update status
- **Reorder** items by dragging
- **Delete** items no longer needed
- **Track progress** via the completion percentage

## Common mitigation activities

Consider these categories when planning:

| Category | Examples |
|----------|----------|
| **Communication** | Announce the change, explain benefits, address concerns |
| **Training** | Conduct workshops, create guides, provide hands-on practice |
| **Support** | Establish help channels, assign champions, create FAQs |
| **Process** | Update procedures, create job aids, revise documentation |
| **Technical** | Configure systems, test integrations, prepare environments |

## Tracking completion

The impact detail page shows:

- **Completion percentage** - Based on completed items
- **Timeline** - Visual representation of start to completion
- **Activity log** - History of changes to the mitigation plan

## Linking to tasks

For complex mitigation activities, you can create formal tasks:

1. From a mitigation item, click **Create Task**
2. The task is created in the Kanban board
3. Task completion updates the mitigation item status

This integrates your mitigation planning with your overall work management.
