Programs help you group related change initiatives together, enabling coordinated planning, consolidated reporting, and better visibility across multiple projects.

## What is a program?

A program is a container for related change initiatives that share:

- Common strategic objectives
- Overlapping stakeholder populations
- Interdependencies or sequencing requirements
- Shared timelines or resources

**Example**: A "Digital transformation" program might include:
- CRM implementation project
- Legacy system retirement
- Process automation initiative
- Digital skills training program

## Programs vs. projects

| Aspect | Program | Project (Initiative) |
|--------|---------|---------------------|
| Scope | Multiple related changes | Single change effort |
| Duration | Typically longer-term | Defined start/end |
| Management | Strategic oversight | Tactical execution |
| Reporting | Aggregated metrics | Detailed tracking |

## Benefits of using programs

### Consolidated visibility

See the big picture across related initiatives:

- Combined stakeholder counts
- Aggregate impact summaries
- Cross-project dependencies
- Portfolio-level status

### Coordination

Identify and manage cross-project concerns:

- Stakeholders affected by multiple initiatives
- Communication timing to prevent fatigue
- Resource conflicts
- Sequencing requirements

### Reporting

Generate program-level reports for leadership:

- Overall program health
- Combined risk assessment
- Stakeholder readiness across initiatives
- Timeline coordination
