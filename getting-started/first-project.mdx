Projects are the foundation of change management in Leap Change. This guide walks you through creating your first project and setting it up for success.

## Before you start

Gather information about your change initiative:

- What is changing and why?
- Who will be affected?
- What's the timeline?
- What outcomes do you expect?

## Creating a project

1. Navigate to **Projects** in the sidebar
2. Click **New Project**
3. Enter a project name that clearly identifies the initiative
4. Click **Create**

## Project details

After creation, configure your project:

### Basic information

- **Name**: Clear, descriptive title (e.g., "ERP Implementation - Phase 1")
- **Description**: Summary of the change and its objectives
- **Status**: Active, On Hold, Completed, or Archived

### Timeline

- **Start date**: When work begins
- **Target date**: Expected completion
- **Phase**: Current project phase

## Adding stakeholders to the project

Link stakeholders who are affected by or involved in this change:

1. Go to the **Stakeholders** tab within your project
2. Click **Add Stakeholders**
3. Search and select from existing stakeholders, or create new ones
4. Set their project-specific roles (if applicable)

<Tip>
  You can also add stakeholders from the main Stakeholders page by selecting them and choosing **Assign to Project**.
</Tip>

## Recording change impacts

Document what will change:

1. Go to the **Change Impacts** tab
2. Click **Add Impact**
3. Describe the current state vs future state
4. Set severity (High, Medium, Low)
5. Link affected stakeholders

## Project status tracking

Track your project's health with:

| Metric | What it shows |
|--------|---------------|
| Stakeholder count | People linked to this project |
| Impact count | Change impacts identified |
| Open surveys | Active feedback collection |
| Readiness score | Average stakeholder readiness |

## Project team

Add team members who work on this project:

1. Go to project settings
2. Click **Add Team Member**
3. Select organization members
4. Assign project roles

## Linking to a program

If this project is part of a larger initiative:

1. Open project settings
2. In the **Program** field, select the parent program
3. Save changes

This enables program-level reporting and coordination.
