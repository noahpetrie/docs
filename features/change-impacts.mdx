Change impacts are the specific effects your initiative will have on people, processes, technology, and the organization. Systematically managing these impacts is essential for successful change delivery.

## Understanding change impacts

A change impact describes:

- **What** is changing
- **Who** is affected
- **How severe** the impact is
- **What actions** are needed to address it

## Creating change impacts

### Manual entry

1. Navigate to **Change impacts** in the sidebar
2. Click **Add impact**
3. Fill in the impact details:

| Field | Description |
|-------|-------------|
| Title | Brief summary of the impact |
| Initiative | Which project this impact belongs to |
| Category | Type of impact (people, process, technology, organization) |
| Severity | How significant the impact is |
| Status | Current state of addressing the impact |
| Current state | How things work today |
| Future state | How things will work after the change |
| Description | Detailed explanation of the impact |
| Mitigation plan | Actions to address the impact |
| Affected stakeholders | Who is impacted |

4. Click **Save**

### AI-assisted impact creation

Use the AI assistant to help identify impacts:

1. Go to **AskLeap** in the sidebar
2. Describe your change initiative
3. Ask for help identifying potential impacts
4. Review and add suggested impacts to your registry

## Impact categories

### People impacts

Changes affecting individuals:
- Role or job changes
- New skills required
- Behavioral changes
- Workload changes
- Reporting line changes

### Process impacts

Changes to how work gets done:
- New procedures
- Modified workflows
- Policy changes
- Compliance requirements

### Technology impacts

Changes to systems and tools:
- New applications
- System migrations
- Interface changes
- Data changes

### Organization impacts

Structural changes:
- Team restructuring
- Location changes
- Departmental changes
- Operating model changes

## Severity levels

<AccordionGroup>
  <Accordion title="Critical">
    Major disruption expected. Requires immediate attention and dedicated resources. May require executive involvement.
  </Accordion>
  <Accordion title="High">
    Significant impact on operations or people. Needs careful planning and active management.
  </Accordion>
  <Accordion title="Medium">
    Moderate impact that can be managed with standard change management practices.
  </Accordion>
  <Accordion title="Low">
    Minor impact requiring minimal intervention. Standard awareness activities sufficient.
  </Accordion>
</AccordionGroup>

## Impact workflow

Track impacts through their lifecycle:

<Steps>
  <Step title="Backlog">
    Impact identified but not yet being addressed
  </Step>
  <Step title="Todo">
    Scheduled to be addressed in the current planning cycle
  </Step>
  <Step title="In progress">
    Actively being managed with mitigation activities underway
  </Step>
  <Step title="In review">
    Mitigation complete, being evaluated for effectiveness
  </Step>
  <Step title="Done">
    Successfully addressed and closed
  </Step>
</Steps>

Impacts can also be marked as **Canceled** or **Duplicate** when appropriate.

## Bulk operations

Manage multiple impacts efficiently:

1. Select impacts using checkboxes
2. Use the bulk action menu to:
   - Change status for all selected
   - Update severity
   - Delete impacts

## Linking to stakeholders

Connect impacts to the people they affect:

1. Open an impact
2. In **Affected stakeholders**, select relevant people or groups
3. Save the changes

This enables:
- Stakeholder-centric impact views
- Targeted communication planning
- Change burden analysis

## Impact review workflow

For detailed impact assessment:

1. Open an impact and click **Review**
2. Work through the review checklist:
   - Confirm severity assessment
   - Validate stakeholder identification
   - Review mitigation plan
   - Document assumptions and risks
3. Mark the review as complete

## Mitigation planning

Document how you'll address each impact:

- **Actions**: Specific tasks to mitigate the impact
- **Owner**: Who is responsible
- **Timeline**: When actions will be completed
- **Status**: Progress tracking

<Tip>
  Link mitigation actions to your communication plan to ensure stakeholders are informed about support activities.
</Tip>

## Impact reports

Access impact analytics from **Reports**:

- Impact distribution by category and severity
- Stakeholder impact summary
- Progress tracking over time
