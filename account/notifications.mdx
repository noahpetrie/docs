Configure notification preferences to stay informed without being overwhelmed.

## Accessing notification settings

1. Click your avatar in the top navigation
2. Select **Settings**
3. Click **Notifications** in the sidebar

## Notification channels

### In-app notifications

Notifications appear in the notification center (bell icon in top navigation):

- Always enabled for critical updates
- Configure which events trigger notifications
- Mark as read individually or all at once

### Email notifications

Receive notifications via email:

| Setting | Description |
|---------|-------------|
| **Immediate** | Email sent as events occur |
| **Daily digest** | Summary email once per day |
| **Weekly digest** | Summary email once per week |
| **Off** | No email notifications |

## Quiet hours

Set times when notifications are paused:

1. Enable **Quiet Hours**
2. Set start and end times
3. Choose which days apply
4. Select timezone

During quiet hours:

- In-app notifications are still delivered but silently
- Email notifications are held until quiet hours end
- Urgent notifications (if configured) can override

## Managing notifications

### Notification center

Click the bell icon to view recent notifications:

- **Unread** - Highlighted notifications
- **All** - Complete notification history
- **Mark all read** - Clear unread status

### Individual notification actions

| Action | Result |
|--------|--------|
| Click | Go to related item |
| Mark read | Remove highlight |
| Dismiss | Remove from list |

## Team-wide defaults

Organization admins can set default notification preferences for new members:

1. Go to **Settings** → **Organization** → **Defaults**
2. Configure notification defaults
3. New members inherit these settings

Individual members can still customize their own preferences.
